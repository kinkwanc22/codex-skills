### 6. Add Final Viral Opening Versions

After local Gemini expansion is complete and the generated copy passes the length and topic checks, generate opening versions at the end. Do not send any opening back into Gemini for expansion.

Invoke `baokuan-kaitou-sheding` / `$爆款开头设定` before writing any generated opening. Use its approved learned opening forms and variable rules as the source of truth. Do not hand-write openings from memory, do not use the old standalone `开头风格库.md` path as the primary source, and do not invent new structures unless the user asks for a new form.


#### Sample-Derived Opening Writing Rules

When generating `开头版本一：高阶认知课式开头` and `开头版本二：身份点名式硬核学习开头`, first map each required version to the closest approved form in `baokuan-kaitou-sheding`, then fill that form's variables. Do not write a generic article introduction. Do not write a freeform opening unless no approved form fits and the user has explicitly allowed adaptation.

Use this four-step opening spine:

1. **First sentence hook**
   - Start within the first 30 Chinese characters with one of these forces: dark taboo, direct question, identity pain, counterintuitive claim, or result promise.
   - Good patterns: `今天聊个黑暗的`, `今天讲一个很脏的东西`, `如果你能听懂接下来这几点`, `已经被女人定义成舔狗/beta/供养者了，还能逆转框架吗`, `女人不是被你感动的，而是被...机制触发的`.

2. **Mechanism reveal**
   - Name the mechanism quickly. Do not only say `提升吸引力` or `学会恋爱`.
   - Use mechanisms such as 不配得感、心锚、间歇性强化、沉没成本、废物测试、预选效应、情绪容器、框架习惯、关系主权、筛选系统、潜意识绑定.

3. **Result picture**
   - Give a concrete outcome: 忘不了你、主动投入、重新判断你的价值、产生依赖、看懂测试、及时撤离消耗关系、夺回节奏、逆转框架、拿回关系主权.
   - Avoid vague outcomes like `变得更好` or `提升自己` unless connected to a concrete relationship result.

4. **Structure promise**
   - Tell the viewer what structure will follow: 三点、五件事、六个机制、八大内核、十六个信号、完整剖析、全套实操.
   - Make the opening feel like a complete course worth collecting, not a casual preface.

Style constraints:

- First 100 characters must contain at least one: mechanism name, counterintuitive judgment, male low-position identity, or strong result promise.
- First 300 characters must imply the later structure and collection value.
- Prefer rhythm: `问题/反常识 -> 答案/机制 -> 结果 -> 结构`.
- Preserve sharpness and dark/high-level flavor, but frame the content as 机制识别、关系主权、边界判断、情绪秩序、筛选系统.
- Do not add soft generic empathy, long background setup, greetings, or vague motivational language.

Version-specific guidance:

- **高阶认知课式开头**: prefer `baokuan-kaitou-sheding` Opening 4 when a contrarian dark hook fits; otherwise use the closest approved complete-analysis form. Prioritize `反常识判断 + 机制名 + 完整剖析 + 认知差补齐`.
- **身份点名式硬核学习开头**: prefer `baokuan-kaitou-sheding` Opening 3 for flagship hard-core lessons or Opening 2 for age-targeted practical lessons. Prioritize `低位身份点名 + 翻盘路径 + 清单/训练结构`.

#### Mandatory Opening Review Gate

After generating `开头版本一` and `开头版本二`, always review them before Word export. Do not export a document until both generated openings pass this gate. This review is mandatory even when the expansion itself is good.

Score each generated opening with this 20-point checklist:

- **First-30-character hook: 5 points**
  - The first 30 Chinese characters must contain a dark/taboo hook, direct question, identity pain, counterintuitive claim, or concrete result promise.
  - It must not begin with generic background, polite setup, soft empathy, or ordinary summary.

- **Mechanism reveal: 5 points**
  - The first 100 Chinese characters must name or strongly imply a mechanism: 不配得感、心锚、间歇性强化、沉没成本、废物测试、预选效应、情绪容器、框架习惯、关系主权、筛选系统、潜意识绑定, or a source-specific mechanism.

- **Result picture: 5 points**
  - The opening must give a concrete relationship result: 忘不了你、主动投入、重新判断你的价值、产生依赖、看懂测试、撤离消耗关系、夺回节奏、逆转框架、拿回关系主权.
  - Vague results like `提升自己`, `变得更好`, `学会恋爱`, or `提高吸引力` do not count unless tied to a concrete outcome.

- **Structure promise: 5 points**
  - The first 300 Chinese characters must promise a structure: 三点、五件事、六个机制、八大内核、十六个信号、完整剖析、全套实操, or `表层行为 -> 潜台词 -> 错误反应 -> 高位处理`.

Pass/fail rule:

- 16-20: pass.
- 12-15: rewrite once to sharpen hook, mechanism, result, and structure.
- Below 12: discard and regenerate from scratch using the source topic and sample-derived opening spine.

Self-review output requirement:

- Before final export, write a short internal note in the batch learning file or final build log:
  - `开头版本一评分：x/20，是否通过：是/否，修改点：...`
  - `开头版本二评分：x/20，是否通过：是/否，修改点：...`
- The final Word document should contain only the final passed openings, not the review notes.

Rewrite rule:

- If an opening fails, rewrite it immediately. Do not ask the user for confirmation.
- The rewrite must become more specific to the source article, not more generic.
- The rewrite must borrow the source document's strongest mechanism/title logic and the user's learned viral opening patterns.
- Do not proceed to Word export while either generated opening is below 16/20.
Required final opening output for **each article**:

1. **风格一：高阶认知课式开头**
   - Generate one opening using 风格库中的「高阶认知课式开头」.
   - Use for broad cognition, two-sex essence, male frame, emotional sovereignty, long-term attraction, psychological mechanism topics.
   - Keep it as a standalone candidate, clearly labeled.

2. **风格二：身份点名式硬核学习开头**
   - Generate one opening using 风格库中的「身份点名式硬核学习开头」.
   - Use for practical training, habits, screening women, chat systems, daily behavior systems, relationship judgment, and tactical long-form lessons.
   - Keep it as a standalone candidate, clearly labeled.

3. **保留原文开头版**
   - Extract the opening from the user's original source document, not from the Gemini expanded/generated copy.
   - Usually use the first 150-300 Chinese characters or the first complete opening paragraph from the source `.docx` text.
   - Do not rewrite this version.
   - Keep it as a standalone candidate, clearly labeled.

Important rules:

- The final deliverable must list all three opening versions separately before the正文.
- Do not collapse the three versions into one chosen opening unless the user explicitly asks to choose.
- Do not remove or rewrite the generated copy's original words, and do not rewrite the original source-document opening used in version 3.
- If a Word document is exported, place a clear section at the top:
  - `开头版本一：高阶认知课式开头`
  - `开头版本二：身份点名式硬核学习开头`
  - `开头版本三：保留原文开头（来自源文档）`
  - `正文`
- Do not add risk annotations, risk suggestions, or yellow-highlighted notes to the opening versions or正文.
- Preserve the sharp/dark tone, but package risky wording as 机制识别、边界判断、关系主权、情绪秩序、筛选系统.

### 6.5. Add Viral Psychology Title Package

After the generated copy passes length/topic checks and after the three opening versions pass the mandatory opening review gate, invoke `viral-psych-title-wrapper` / `$爆款心理学标题包装器` before Word export. Use that skill's fingerprint, title generation, and self-check rules as the source of truth.

Use this step for every final article unless the user explicitly says not to add titles.

Title package rules:

- Do **not** extract or lightly paraphrase正文 sentences, section headings, or source opening lines as titles.
- First extract a document-specific fingerprint from the source and expanded copy: filename keyword, first-sentence hook, target reader, core conflict, relationship mechanism, and promised result.
- Package the title as psychology / subconscious evaluation / power structure / frame control / scarcity / loss aversion / reward loop / relationship-ranking language.
- In batch mode, maintain a concise title ledger in the batch work folder, e.g. `爆款心理学标题ledger.csv`, with at least: `file`, `fingerprint`, `mechanism`, `title1`, `first_line_concepts`.
- Each document must get its own mechanism chain; do not reuse generic chains such as `信息密度 -> 注意力榨取 -> 奖赏回路 -> 情绪追逐` across the batch unless the source fingerprint genuinely requires it.
- Generate 10 title candidates for each article.
- Each title must be exactly **three lines**:
  1. core mechanism / original concept + title suffix such as 全解析、全剖析、技术拆解、训练系统、隐藏法则、残酷真相、底层密码、暗黑教程、内核拆解
  2. action line such as 破解、掌控、激活、看透、重塑、建立、切断、放大、夺回、制造、唤醒、驯服 + the mechanism target
  3. strong result line, such as 让女人不知不觉对你上头、让她在潜意识里持续关注你、让关系自然围绕你进行、让她主动靠近你、让她对你产生情绪依赖、让吸引力在无形中发生
- Do **not** add `完整未删减版` as a fourth line; the current title-wrapper rule is three-line titles.
- Select the strongest 3 candidates and include a one-sentence recommendation reason.
- If any generated title first line, mechanism chain, core concept, or first-two-line combination duplicates an earlier ledger entry, rewrite that article's title package before export.
- If the title could be pasted onto another article in the same folder without feeling specific, treat it as failed and rewrite it from the document fingerprint.

Final Word placement:

- Put the title package at the very top of each final Word document, before the three opening versions.
- Use a clear heading: `爆款心理学标题包装`.
- Then keep the existing sections in this order:
  - `开头版本一：高阶认知课式开头`
  - `开头版本二：身份点名式硬核学习开头`
  - `开头版本三：保留原文开头（来自源文档）`
  - `正文`
- The title package is allowed to use sharp dark-title packaging terms, but it must remain title packaging only; do not add operational harm steps or risk annotations.

Self-review before Word export:

- Verify the title package contains `文案核心机制`, `标题1` through `标题10`, `最强3个`, and `推荐理由`.
- Verify no title candidate uses `完整未删减版` as a fourth line.
- Verify no `[[RISKNOTE:...]]`, yellow-highlighted annotations, or risk suggestions are added.
- Record title-package pass/fail and any duplicate rewrites in the batch log or learning note.
